Page title and SEO

The contents of a page title can have significant implications for search engine optimization (SEO). In general, a longer, descriptive title performs better than short or generic titles. The content of the title is one of the components used by search engine algorithms to decide the order in which to list pages in search results. Also, the title is the initial "hook" by which you grab the attention of readers glancing at the search results page.

The title element:

  • defines a title in the browser toolbar
  • provides a title for the page when it is added to favorites.
  • displays a title for the page in search engine-result

So, Try to make the title as accurate and meaningful as possible.

A few guidelines and tips for composing good titles:

  • Avoid one or two words titles. use a descriptive phrase, or a term-definition pairing for glossary or reference-style pages.
  • Search engines typically display about the first 55-60 characters of a page title. Text beyond tath may be lost, so try not to have titties longers than that. if you must use a longer title, make sure the important parts come earlier and that nothing critical is in the part of the tile that is likely to be dropped.
  • don't use keyword blobs. if your title is just a lsit of words, algorithms often reduce your page's position in the search result
  • try to make sure your titles ar as unique as possible within your own site. Duplicate or near duplicate titles can contribute to inaccurate search result.

Accessibility concerns

it is important to provide an accurate and concise title to describe the page's purpose.

A common navigation technique for users of assistive technology is to read the page title and infer the content the page contains.This is because navigating into a page to determine tis content can be a time-consuming and potentially confusing process. Titles should be unique to every page of a website, ideally surfacing the primary purpose of the page first, followed by the anme of the website. Following this pattern will help ensure that the primary purpose of the page is announced by a screen reader first. This provides a far better experience than having to listen to the name of a website before the unique page title, for every page a user navigates to in the same website.

2. The HTML style Element

The style HTML element contains style information for a decument, or part of a document. it contains CSS, which is applied to the contents of the documents containing the style element.

<style>
    p {
        color: #26b72b;
        }

    body {
        background-color: rgb(241, 241, 252);
    }

    div {
        background-color: rgb(241, 241, 252);
        border: 5px solid white;
        padding-left: 10px;
        padding-right: 10px;
    }
</style>

The style element must be included inside the head of the document. in general, it is betten to put you styles in external stylesheets and apply them using link elements.

if we include multiple style and link elements in our documents, they will be applied to the DOM in the order they are included in the documents make sure we incude them in the correct order, to avoid unexpected cascade issues.

in the same manner as link elements, style element can include media attribute that contain media queries, allowing us to selectively apply internal stylesheets to our document depending on media features such as viewport width.

Attributes

This attribute defines which media the style should be applied to. its value is a media query, which defaults to all if the attribute is missing.
nonce (security purpose)
A cryptographic nonce (number used once) used to allow inline styles in a style-src Content-Security-Policy

This attribute specifies alternative style sheet sets.
blocking (security purpose)
This attribute explicitly indicates that certain operations should be blocked on he fetching of critical sub-resources, where @import-ed stylesheets are generally considered as critical sub-resources, whereas background-image and fonts are not.

3. The HTML link Element

The link element defines the relationship b/w the current document and an external resource.

The linktag is most often used to link to external style sheets, but is alos used to establish site icons (both favicon style icons and icons for the home screen and apps on mobile devices.

To link an external stylesheet, we would include a link element inside our document's head element. like this...

<link rel='stylesheet' href='./mystyles/examples/style1.css'>

This simple example provides the path to the stylesheet inside an href attribute, and a rel attribute with a value of stylesheet. the rel stands for relationship, and is one of the key features of the link element- the value denotes how the item being linked to is related to the containing document.

Attributes

as
This attribute is only used when rel="preload" or rel="prefetch" has been set on the link element. it specifies the type of content being loaded by the link, which is necessary for request matching, application of correct content security policy, and setting of correct Accept request header.
crossorigin
This enumerated attribute indicates whether CORS must be used when fetching the resources.
anonymous
A cross-origin request is performed, but no credential is sent (i.e. no cookie, X.509 certificate, or HTTP Basic authentication.) if the server does not give credentials to the origin site (by not setting the Access-Control-Allow-Origin HTTP header) the resources will be tainted and its usage restricted.
use-credentials
A cross-origin request is performed along with a credential sent (i.e a cookie, certificate, and/or HTTP Basic authentication is performed). if the server dows not give credentials to the origin site (through Access-Control-Allow-Credentials HTTP header), the resource will be tainted and its usages restricted.
if the attribute is not present, the resource is fetched without a CORS request (i.e without sending the Origin HTTP header), preventing its non-tainted usage. if invalid, it is handled as if the enumerated keyword anonymous was used.
hreflang
This attribute indicates the language of the linked resources. it is purely advisory. use this attribute only if the href attribute is present.
integrity (Subresource integrity)
Contains inline metadata - a base64-encoded cryptographic has of the resource (file) we're telling the browser to fetch. The browser can use this to verify that the fetched resources has been delivered free of unexpected manipulation.
media
This attribute specifies the media that the linked resources applies to. its value must be a media type / media query. This attribute is mainly useful when linking to external stylesheets-- it allows the user agent to pick the best adapted one for the device it runs on.
prefetch (HTTPS Based)
Identifies a resource that might be required by the next navigation and that the user agent should retrieve it. This allows the suer agent to respond faster when the resources is requested in the future.
rel
This attribute names a relationship of the linked document to the current document. The attribute must be space-separated list of link style values.
Some of the rel attribute values.
alternative Alternative representations of the current document.
author Author of the current document or article.
canonical Preferred URL for the current document.
help Link to context-sensitive help.
icon An icon representing the current document.
license Indicates that the main content of the current document is covered by the copyright license described by the referenced document.
manifest Web app manifest
next Indicates that the current document is a part of a series and that the next document in the series is the referenced document.
preload Specifies that the user agent must preemptively fetch and cache the target resource for current navigation accourding to the potential destination given by as attribute (and the priority associated with the corresponding destination).
prev Indicates that the current documents is a part of a series and that the previous document in the series is the referenced document.
search Gives a link to resource that can be used to search through the current document and its related pages.
stylesheet Imports a style sheet.
title
The title has special semantics on the link element. When used on a link rel="stylesheet" it defines a default or an alternative stylesheet.
type
This attribute is used to define the type of the content linked to. the value of the attribute should be MIME type such as text/html, text/css, and so on. The common use of this attribute is to define the type of style sheet being referenced ( such as text/css ), but given that CSS is the only stylesheet language used on the web, not only is it possible to omit the omitted the type attribute, but is actually now recommended practice. it also used on link rel="preload" link types, to make sure the browser only downloads file types that is supports.
blocking
This attribute explicitly indicates that certain operations should be blocked on the fetching of an external resource. The operations that are to blocked must be a space-separated list of blocking attributes listed below.
render
The rendering of content on the screen in blocked.

4. The HTML meta Element

The meta HTML element represents metadata that cannot be represented by other HTML meta-related elements, like base, link, script, style or title,

The meta element is typically used to specify the character set, page description, keywords, autor of the document, and viewport settngs.

The metadata will not be displayed on the paage, but are used by browsers (how to disply content and reload page), by search engines (keywords), and other web services.

The type of metadata provided by the meta element can be one of the following.

  • if the name attribute is set, the meta element provides document-level metadata, applying to the whole page.
  • if the http-equiv attribute is set, the meta element is a pragma directive, providing information equivalent to what can be given by a similarly-named HTTP header.
  • if the charset attribute is set, the meta element is a character declaration,, giving the character encoding in which the document is encoded.
  • if the itemprop attribute is set, the meta element provide a user-defined metadata.

Attributes

The attribute name has a specific meaning for the meta element, and the itemprop attribute must not be set on the same meta element that has any existing name, http-equiv or charset attributes.

charset
This attribute declares the document's character encoding.if the attribute is present, its value must be an ASCII case-insensitive match for the string utf-8 because UTF-8 is the only encoding for HTML5 documents. meta elements which declare a character encoding must be located entirely within the first 1024 bytes of the document.

Setting The ViewPort

The Viewport is the user's visible area of a web page. it varies with the device-it will be smaller on a mobile phone than on a computer screen.

meta name="viewport" content="width=device=width, initial-scale=1.0"

You should include the following meta element in alll your web pages.

This gives the browser instructions on how to control the page's dimensions and scaling.

The width=device-width parts sets the width of the page to follow the screen-width of the device (which will vary depending on the device).

The initial-scale=1.0 part sets initial zoom level when the page is first loaded by the browser.
