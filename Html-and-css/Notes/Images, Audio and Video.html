Images, Audio and Video

In this section we will learn how to add images, animations, audio and video to our web page or site. image, audio and video are the other type of content which allow us to share the data in the visual and audible form. In all them images are used to increase the visual effect of our web page or site.

we put an image on the web page by using the img which has no closing tag. Images can be in different format, generally we use three kind of image format to put them on the web page. these are (JPEG, GIF, PNG)

we put the audio and video by using the object, param, embed elements.

Adding Images Using the img Element

Images are added to a site by using the img elmement, which has to carry at least tow attributes: the src attribute, indicate the source of the image, and an alt attribute, which provide a description of the image.

For example we will put an image google logo.

img src="google-128px.ico" alt="Google Home Page"

The img element tag can carry all the universal attributes and the UI event attributes. The img element can carry the following attributes:

Attributes: src, alt, align, border, height, width, hspace, vspace, ismap, usemap, longdesc, name

The src Attributes

The src attribute tells the borwser where to find the image. The value is a URL.

We should always maintain our image at our server and should avoid the using the images from different site. images form different sites may delete or moved from one location to another, in this case we will be able to display the imgae on our page. In best practice we should always store the image on the our server side and use them instead of external image.

The alt Attributes

The alt attribute must appear on every img element and its value should be a text description of the image.

img src="image_file_path" alt="image description"

The image description should be related to the image since:

  • if the browser cannot dispaly the image, this text alternative will be shown instead.
  • Web users with visual impairments often use software called a screen reader to read a page to tem, in which case the alt text descrbes the images they cannot see.
  • While search ensgines are very clever, they cannot yet describe or index the contents of an image; therefore, provididng a text alternative helps search engines index your pages and helps visitors find your site.

The height and width Attributes

The height and width attributes specify the height and width of the image, and the values for these attributes are almost always shown in pixles.

img src="google-128px.ico" alt="Google" width="64" height="64"

we can use other units to set the image width and height for example we can use the percentage value, it will take the the specified percentage space of the it's parent container.

Note: if we pass the only one value either width or height it will maintain the aspect ratio other wise it will distort (damage the image quality) the image.

Image Maps

Image maps allows us to add multiple links to the same image, with each link pointing to a different page. They are particularly helpful when thhe image needs to be divided up in irregular shapes (such as maps). Each of these clickable areas is known as a hotspot.

Image maps works is based on upon x and y coordinates, measured from the top left-hand corner of the image. These coordinate are used in two ways.

  • To specify where the hotspots are.
  • To work out where the user is clickng on the image.

By Comparing the coordinates of the hotspots with the coordinates where the user clicks, the user can be sent to the link specified for that hotspot.

There are actually tw types of image maps, both of which do the same job. The difference is how they work out, which hotspot you clicked on.

  • Client-side image maps: The web browser works out which part of the image the user has clicked on and takes the user to the appropriate page.
  • Server-side image maps: The browser sends the sever the coordinates, saying which part of the image the user clicked on, and these are processed by a script file on the server that determines which page the user should be sent to.

Client-Side Image Maps

Client-Side image maps use HTML code to tell the browser how the image will be divided up; then the browser can tell what part o fthe image someone clicks on and can send the user to the appropriate link.

The are two methods to create a client-side image map:

  • Using the map and area elements inside an img element.
  • Using the map element inside the object element.

Client-Side Image Maps Using map and area

    Process to make image map
  1. To Make Client-Side Image map, first of all we will insert our image into our page using the img element.
  2. An attribute called usemap is then added to the img element to indicate that this image need to use the information held in the corresponding map element to turn the image into an image map.
  3. The map element than conatin the information used to tell the browser how to split up the image so that it can send the user to the correct page.
  4. The value of the usemap attribute begins with a pound or ahs sign, followed by the value of the name.
  5. The map element usually follows directly after the img element. it needs to carry onlty the name attribute to identify itself.
  6. The map element then contain two or more area elements, and it is these area elements that actually define the clickable hotspot.
  7. The area element does this by specifying the shape and the coordinates that define the boundaries of each clickable hotspot.

The attributes that the area element may have the following attribues:

Attributes: acesskey, alt, shape, coords, href, nohref, target, tabindex, taborder, notab

The area Element Attributes

The shape Attribute

The value of the shape attribute actually affects how the browser will use the coordinates specified in the coords attribute, and is therefore required. if we don't specify a shape attribute, the default value in a rectangle.

Values of shape attribute: default, rectangle or rect, polygon or poly, circle or circ

The value default should be used on the last area element if we want to indicate any sections of the image not otherwise indicated by an area element that has defined an area.

The coords Attribute

The coords attribute specifies the area that is the clickable hotspot. The number of coordinates we specify depends on the shape we are creating.

  • A rectangle contains four coordinates. The first two coordinates represent the top left of the rectangle, and the second two the bottom right.
  • A circle contains three coordinates; the first two are the center of the circle, while the third is the radius in pixles.
  • A polygon contains two coordinates for each point of the polygon. So a triangle would contain six coordinates, a pentagon would contain ten, and so on. We don't need to specify the first coordinate again to close the geomatry at the end because the shape is automatically closed.

The href and nohref Attributes

The href attribute works just like the href attribute for a element; its value is the URL of the page we want to load when the user clicks that part of the image.

if we don't want part of the image to link anyway; we should use the nohref attribute on that space to indicate that the area will not take us anywhere.

if we want to use the nohref then we should give it a value as "nohref".

area shape="rect" coords="234, 234, 232, 423" nohref="nohref"

The alt Attribute

The alt attribute specifies a text alternative for that section of the image and works just like the alt attributes on the img element.

It will acutally override the alt text specified for the image when the user rolls over the area.

The target Attribute

The target attribute specifies which frame or window the page should be loaded into. Possible values are same as for the target attribute of the a element

The tabindex Attribute

The tabindex attributes allows us to specify the order in which users can tab through items on a page. The value is a number b/w the 1 and 32,767.
