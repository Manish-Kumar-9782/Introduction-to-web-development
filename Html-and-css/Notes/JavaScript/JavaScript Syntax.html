JavaScript Syntax

Every programming follows some rule to process the code, this rule is called syntax, in this section we will see the basic Syntax in which we will see that what are the rule should be in our mind while writing a JavaScript code.

Whitespace

Whitespace refers to characters that provide the space b/w other character. (space in b/w the words) JavaScript has the following Whitespace.

  • Carrage return
  • Space
  • New Line
  • tab

JavaScript engine ignores whitespace. However, we can use whitespace to format the code the code to make it easy to read and maintain.

following code doesn't use must whitespace

code is written in single line.

var a = 10; var b = 20; var c = a+b;

This above code can be written in the following format. which is easy to read.

A code with formatting by using the whitespaces and new lines.

var a = 10;
var b = 20;
var c = a + b;

JavaScript bundlers remove all whitespace from JavaScript files and put them into as single file for deployment. By doing this, JavaScript bundlers make the JavaScript code lighter and faster to load in the web browsers.

Statements

A Statement is a code that declares a variable or instructs the javaScript engine to do a task. A simple statment is terminated by a semicolon (;)

Although the semicolon (;) is optinal; we should always use it to terminate a statment.

below code terminate the statement by using the ';' sign.

let message = "Hello World!";
console.log(message);

blocks

A block is a sequence of zero or more simple statements. A block is delimited by a pair of curly brackets {}.

if (command = 'yes') { console.log("Access Granted"); element = document.getElementById("idname"); element.style = "setting some style"; } else{ console.log("Access Denied"); }

Identifiers

An identifier is a name we choose for variables, parameters, functions, classes, etc. An identifier anme starts with a letter (a-z, or A-Z), an underscore (_), or a doller sign ($ and is followed by a sequence of characters inclyuding ( a-z, A-Z), numbers (0-9), underscores (_), and doller sign ($).

The letter is not limited to the ASCII chracter and many include extended ASCII or Unicode thought not recommended.
Identifiers are case-sensitive. for example, the variable is different from Variable.

JavaScript defines a list of reserved keywords that have specific uses. Therefor, we cannot use the reserved keywords as identifiers or property names by rules.

The follwoing table contains all the reserved identifier which are called keywords. These keywords are defined in ECMA-262.

break case catch
continue debugger default
else export extends

Summary

  • We use whitespace including cariage return, newline, and tab to format the code. The JavaScript engine ignores the whitespace.
  • Use a semicolon : to terminate a simple statement.
  • use the curly braces {} to from a block that group one or more simple statements.
  • A single-line comment start with // followed by a text. A block comment begins with /*and ends with */. javaScript engine also ignores the comments.
  • Identifiers are names that we choose for variables, functions, classes, etc.
  • Do not use the reserved keywords and reserved words for identifiers.
