HTML Core Elements and Attributes

In every HTML Document we have four main elements that form the basic structure of the every HTML Document. these are html head title body.

These four elements should appear in every HTML document to make the HTML basic structure.

The html Element

The html element is the containing element for the HTML document. we should add this this element just after declaring the document type declaration.

Each document should have the opening html and closing tag /html.

In b/w the opening tag and closing tag we need to put the head and body elements.

xmlns attribute

if we have a multiple markup language which are using the same tag name for example big small medium etc. then we need to define to the browser that which tag belongs to which language, in this situation we need to use the xmlns attribute to specify the markup language.

hmtl xmlns="http://www.w3.org//1999/xhtml"


Attributes appllicable on the html are id, dir, lang, xml:lang.

The head Element

A headelement contain the various kind of important information about our document, each html should contain the titleelement indicating the title of the document.

  • base element to define the base address of the page.
  • object element is used to include images, JavaScript Object, flash animation, MP3 Files, QuickTimes movies, and other types of file in a page.
  • link element is used to link to an external file, such as stylesheet, JavaScript etc.
  • style element is used to include the CSS rule inside the current html document.
  • script element is used to include the script in the current document.
  • meta element is used to include the information aobut the document such as keywords and description, which are particularly helpful for search applications like browsers.

    • The head can use these attributes: id, dir, lang, xml:lang

The title Element

We should specify a titlefor every page by using the title element. The use of the can be used in several ways.

  • Title at the tab of the browser (at the very top of the browser).
  • As the default name for a bookmark in many browsers.
  • By Search Engines that use its content to help index pages.

By Considering the importance of the above uses, it is important to use a title that really describes the content of our website.

Title should be short and relative to content of current document.

The title element should contain only the text for the title; it may not contain any other elements.

The title element can have these following attributes: id, dir, lang, xml:lang

The body Element

The body element appears after the head element and as we have already seen, it contain the part of the web page that we actually see in the main brower window, which is referred as body content.

The body element contain many attribute along with styling attributes.

Attributes appllicable on body element: background, bgcolor, alink, link, vlink, text

Attributes Group

Attributes are placed on the opening tag of an element and provide extra information about the element that carries them. All attributes consist of name and value; the name defines the a property of that element and the value is a vlaue of that property.

→ For e.g. xml:lang attribute describe the language used within that element.

  • Core Attributes: The class, id and title attributes.
  • Internationalization attributes: The dir, lang and xml:lang attributes.
  • UI events: Attributes associated with events such as onclick, ondoubleclick, onmousedown, onmouseup, onmouseover, onmousemove, onmouseout, onkeypress, onkeydown, and onkeyup.

Note: Together, the core attriubtes and the internationalization attributes are known as universal attributes.

Core Attributes

The four core attributes that can be used on the majority of HTML elements are: code, title, class, style

The id Attribute

The id attribute can be used to uniquely identify any element within a page. we might want to uniquely identify an element so that we can link to that specifc part in the doucment, or to specify that a CSS style or piece of JavaScript should apply to the content of just that one element within the document.

syntax: id="string"

Rules to apply the id:

  • id name should begin with a letter (A-Z or a-z) and can then be followed by any number of letters, digits, hypens, underscores, colons, and periods.
  • id remain unique within that document; no two id attribute may have the same value within one HTML page.

The class Attriubte

The class attributes is used to specify that an element belongs to a class of elements.

For e.g. we might have a document that contains many paragraphs, and a few of those paragaraphs might contain a summary of key points, in which case you could use add a class attribute whose value in summary to the relevant p elements, to differentiate those paragraphs from the res in the document.

syntax: class="classname"

syntax: class="classname1 classname2 classname3 ... classnamen"

The value of the class attributes amy also be space-separated list of class names.

The title Attribute

The title attribute gives a suggested title for the element. The syntax for the title attribute is follows:

title="string"


The behaviour of title attribute will depend upon the element that carries it, although it is often displayed as a tooptip or while the element is loading.

Note: Not every element that can carray a title attribute really need one, it should only used when required.

The style Attribute

The styleattribute allows you to specify CSS rules within the element.

syntax: style="property1:value; property2:value; ... propertyn:value;"

Since all the CSS rule can be applied by using the style element, so we should avoid to use the style attribute.

We also can use the separate style-sheet which can be applied multiple different HTML document at once and also we don't have to repeate the same CSS rules.

Internationalization

There are three internationalization attributes that helps users write page for different languages and character sets, and they ary available to most HTML elements.

Attributes:dir, lang, xml:lang

Note: The internationalization attributes are sometimes referred to as the i18n attributes, an odd name that comes from the draft-left-html-i18n specification in which they were first defined.

The dir attribute

The dir attribute allows us to indicate to the browser the direction in which the text should flow; left to right or right to left.

when we want to indicate the directionality of a whole document, it should be used with the html element rather than the body element for two reason.

  • → its use on the html element has better support in browsers, and it will apply to the header elements as well as those in the body.
  • → The dir attribute can also be used on element within the body of the document if we want to change the direction of a small portion of the document.

The dir attribute can take one of two values, as we can see in the table that follows.

Value Meaning
ltr Left to right (the default value)
rtl Right to left (for language like Hebrew or Arabic.)

the lang Attribute

the lang attribute allows us to indicate the main language used in a document. has been replaced in newer version of HTML like HTML5 or XHTML

As for good practice we should use both lang and xml:lang attribute on the html element inside the current docuement.

The lang attribute was designed to offer language-specific display to users, although it has little effect in the main browsers.

The read benefits of using the lang attribute are with:

Search Engine:
Search engine can tell the user which language the docuement is authored in.
Screen readers:
Screen reader might need to pronounce different language in differnt ways.
Applications:
Applications can alert users when they either do not support that language or it is a different language than their default language.
With html element:
When it is used with html element the attributes applies to the whole document, although it can be used on the other elements, in which case it just applied to the content of those elements.

The value of the lang attribute are ISO-689 standard two-character language codes. if we want to specify a dialect of the language, we can follow the language code with a dash and a subcode name.

The table shows some example of language code.

Value Meaning
ar Arabic
en English
en-us U.S. English
zh Chinese
hi Hindi

The xml:lang Attribute

The xml:lang attribute is the xHTML replacement for lang attribute. it is an attribute that is available in all language that are writeen in XML, which is why it prefixed by the characters xml:.

The value of the xml:lang attribute should be an ISO-639 country code.

While it has no effect in the main browser, other XML-aware applications and search engines may use this information, and it is good practice to include the xml:lang attribute in our docuemnts.

When used with the html element, it applies to the whole document, although it can be used on other elements, in which case it just applies to the content of those elements.
